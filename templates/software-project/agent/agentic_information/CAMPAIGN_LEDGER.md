# Campaign Ledger

Append-only history for project decisions, releases, migrations, and rejected
approaches. Do not rewrite history except to fix factual mistakes.

## Standing Policies

- Run the quality gate from `AGENTS.md` before finishing substantial work.
- Record nontrivial bug fixes in `CLOSED_LOOP_LEDGER.md`.
- Record architecture, dependency, and workflow decisions here.
- Keep setup and test commands current in `docs/`.
